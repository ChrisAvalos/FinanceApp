# Finance App — Comprehensive Scorecard
**Date:** 2026-05-05
**Method:** Code-based audit. Each panel scored on 7 dimensions
(0-100), then aggregated. Scores reflect source-code state plus
running mental model from this build cycle. Live-UI re-walk not
performed in this pass — last live audit was #269 (2 weeks ago).

---

## Rubric

| Dimension | What it measures |
|---|---|
| **Function** | Does the panel actually work end-to-end? Is the data real? |
| **Polish** | Skeleton states, CountUp on hero stats, no "Loading…" strings, error states, hover/focus details |
| **Ease of use** | Affordances (filter chips, sort, click-to-drill), defaults, copy |
| **Delight** | Toasts, undo, animations, micro-interactions, surprise-and-delight |
| **Beauty** | Visual hierarchy, color use, spacing, typography, density |
| **Efficiency** | Loads fast, useful empty states, can answer the question with minimum clicks |
| **Completeness** | Edge cases handled, onboarding paths, deep-linkable, mobile parity |

Aggregate: simple mean across all 7. Honest grading — 90+ means "I'd
ship this", 70-90 is "useful but rough", below 70 is "exists but
needs work".

## Aggregate: **80.1 / 100** (was ~76 in #269)

The app crossed into "useful but rough" → "polished" territory in
several panels. The lifts on Notifications / Offers / Unclaimed
moved each from the bottom quartile to the top half. The mean is now
held back primarily by the lower-half cluster: Tax, HSA, Holdings,
Redress, CardApps, Benefits, Deals, Canonical Products, Mobile. Most
of those are not broken — they're "wired and functional but
unloved".

   + relative timestamps + clear-read bulk action. Also fixed the

3. **Unclaimed 68 → 86.** CountUp on all 4 hero stats with skeleton

   sort dropdown, per-tab empty state copy.

4. **SmartCredit integration shipped.** TU/EX/EQ VantageScore 3.0
   pulled daily from a single 3B URL; verified live on Chris's account
   (702 / 699 / 707).

## Top remaining gaps (panels < 75)

### DealsPanel (66) — STARTED but not yet shipped
Framework + UI exist, but every store scraper is a stub returning
None. The UX scaffolding (DealsPanel) is decent but the panel reads
hollow because there's no live data flowing.

**To lift:**
- Ship the Target HTTP scraper first (no auth, friendliest endpoint)
- Then Walmart HTTP (similar shape)
- Then Costco / Kroger (Playwright + bootstrap, like the offer scrapers)
- Defer Amazon Fresh (heaviest bot detection)
- Add status strip to DealsPanel mirroring the OffersPanel pattern
- Add click-to-drill from observation → product detail
- CountUp on summary stats

### RedressPanel (66) — never finished
CFPB / state-AG redress feature exists end-to-end but the panel is
visually thin. Filter exists, no skeleton, no empty state, no
CountUp, no status strip.

**To lift:**
- Skeleton + CountUp on hero ($ recovered)
- State filter chips (only when ≥ 2)
- Click-to-drill into source filing
- Better empty-state copy

### CanonicalProductsPanel (67) — utility, not user-facing
Internal-only UI for inspecting how the canonicalizer grouped product
SKUs. Lower priority because the user rarely opens it.

**To lift:** Skeleton, better empty state. Don't over-invest.

### TaxPanel (68), HSA (68), Holdings (68)
All three are "data correct, UI bare." Same generic prescription:
skeleton on load, CountUp on hero stats, better empty states, sort
options where applicable.

### CardApplicationsPanel (71), BenefitsPanel (71), YieldOptPanel (71)
Trio of card-related panels. CardApplications shows 5/24 status
calculation correctly. Benefits shows the per-card unused-benefit
rollup. YieldOpt shows the T-bill arbitrage. All three would benefit
from:
- Skeleton + CountUp pass
- Status timestamp ("rates last refreshed N days ago")
- Better visual hierarchy on the hero numbers

### Mobile (71, broad)
FIRE + Attribution screens are the worst offenders — desktop polish
hasn't propagated to mobile. Specifically:
- FIRE editable steppers are functional but feel clunky on mobile
- Attribution chart on mobile is hard to read
- No skeletons across most mobile screens

## Dimension-level observations

- **Function (avg 86):** the app is real. Almost nothing scores below
  75 here. Live data flows, calculations are right, edge cases mostly
  handled.
- **Polish (avg 75):** the biggest variance — Offers/Notifications/
  Unclaimed/MoT in the 90s, but ~10 panels still in the 60s. This is
  where the next push has the most leverage.
- **Ease of use (avg 81):** generally good — affordances are present
  but inconsistent.
- **Delight (avg 68):** the lowest-average dimension. Undo toasts,
  click-to-drill, animations exist on a handful of panels but not
  most. Adding these is cheap and lifts perceived quality fast.
- **Beauty (avg 76):** Tailwind navy/emerald/amber palette is
  consistent. Typography is plain (system stack). Could be lifted
  with a custom font (Inter would be ~15 minutes) and slightly
  warmer accent palette.
- **Efficiency (avg 81):** mostly good — TanStack Query caching is
  effective. A few panels refetch on every mount unnecessarily.
- **Completeness (avg 79):** edge cases exist; some onboarding paths
  are weak (e.g., no "first time here" tour).

## Roadmap to 90 aggregate

To move the mean from 80 → 90, we need ~75% of panels to score 85+.
Currently 13 of 33 hit that bar. Path:

**Wave A (cheap, highest-impact polish — ~8h):**
1. Skeleton + CountUp pass on the cluster of <75 panels
   (Tax/HSA/Holdings/Redress/CardApps/Benefits/YieldOpt/CanonicalProducts).
   Mostly mechanical adoption of components/Skeleton.tsx + CountUp.tsx.
2. Click-to-drill on Notifications card variants (already in place
   for /api/notifications/{kind}, but other surfaces don't route).
3. Better empty-state copy across the board — write 1-2 sentences
   per panel that explain what this bucket is for.

**Wave B (medium — ~12h):**
4. Cross-store deals: ship Target + Walmart HTTP scrapers, then
   Costco/Kroger Playwright. DealsPanel status strip + activation
   tracking (mirror OffersPanel pattern).
5. Streak counter on Today's Money Moves (delight win).
6. Cmd+K command palette (delight + efficiency).

**Wave C (heavier — ~16h):**
7. Custom typography (Inter, JetBrains Mono for tabular numbers).
8. Mobile FIRE + Attribution polish + skeleton adoption.
9. Hover tooltips on FIRE Monte Carlo chart.
10. Heatmap day-cell click-to-drill into that day's transactions.

After Wave A alone the mean should clear 84. After A+B, ~88.
After A+B+C, ~91-92.

## Methodology caveats

- **Code-based scoring is overconfident on Function** and
  underconfident on Beauty/Delight (the two dimensions you actually
  see in 5 minutes of using the app). Treat this as a planning
  document, not a tournament leaderboard.
- The prior audit (#269) was live-screenshot-based and is the better
  baseline for "would a stranger be impressed". The Δ column reflects
  *my best guess* of how each score has moved given changes since
  then.
- Live re-walk recommended before declaring 90 — many of these scores
  are within 5 of each other and code-only review can't disambiguate
  at that resolution.
