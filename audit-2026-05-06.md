# Finance App — Comprehensive Audit
**Date:** 2026-05-06 (post-typography + Cmd+K + FIRE-tooltip + dedup-prevention shipped)
**Method:** Code-based scoring across 6 dimensions, augmented by browser-MCP visual checks where the dev server is responsive. Compared against the prior audit (2026-05-05).

---

## Rubric — 6 dimensions, 0-100 each

| Dimension | What it measures |
|---|---|
| **Functionality** | Does it work end-to-end with real data? Are calculations correct? Edge cases handled? |
| **UX** | Affordances, copy, defaults, click-to-drill, filter/sort, navigation, keyboard accessibility |
| **Delight** | Skeletons, CountUp animations, undo toasts, streaks, click feedback, micro-interactions, surprise-and-delight |
| **Beauty** | Typography (Inter now ✓), color palette consistency, spacing, density, hierarchy, visual coherence |
| **Intelligence** | Smart defaults, context awareness, inferred state, autonomous detection (anomaly, dedup, categorization rules), AI integration |
| **Completeness** | Empty states, loading states, error handling, mobile parity, deep-linkable, edge-case coverage |

Aggregate per panel = mean of the 6 dimensions. Honest grading — 90+ means "I'd ship this", 75-90 is "useful but rough", below 75 is "exists but needs work".

## Where each dimension stands

### Functionality — 89 (very strong)
Almost every panel works end-to-end with real data. The two functionality drags are DealsPanel (anti-bot blocked retailer scrapers) and a sprinkling of edge cases (e.g., Merchant search silent on no-match — now fixed). Core financial calculations have been validated against multiple methods (FIRE Monte Carlo seasoned with historical starting years, attribution split between debt-paydown and spending, etc.). The Plaid sync is robust including the orphan-account re-link bug fix and the new fuzzy-dedup prevention layer.

### UX — 86 (strong, room to grow)
Cmd+K command palette just shipped — major affordance win. Most panels have filter chips, sort options, and click-to-drill where appropriate. Empty-state copy is uniformly good after the Wave A push. Two remaining UX gaps:
- **Onboarding tour for new users.** No guided "first 5 minutes" experience — users are dropped into Overview cold.
- **Bulk actions on Transactions** are present but not discoverable; no contextual hint that they exist.

### Delight — 80 (was 68, biggest jump)
Skeletons across 16 panels. CountUp on hero stats across 14 panels. Undo toasts on Notifications. Streak chips on Daily Moves. Click-to-drill from Notifications kind → source panel. Hover tooltips on FIRE chart. Pie-slice expansion + dim on Trends. Per-portal "I don't have this" hide-button on Offers. There's still room — Cmd+K could trigger an entrance-animation, the Daily Moves "✓ Done" click could play a satisfying micro-animation, and a confetti moment on hitting a goal milestone would be earned — but the floor has moved up dramatically.

### Beauty — 88 (was 76, second-biggest jump)
Inter as the body font is a step-change. JetBrains Mono on tabular numbers gives them proper weight. The navy/emerald/amber/rose Tailwind palette is consistent. Card shadows are subtle. The Trends pie chart with proper distinct colors (12-color palette + slate "Other" overflow) shows real visual coherence. Heatmap's GitHub-style grid is iconic. The FIRE fan plot reads cleanly.

What would push this past 90: a real type scale revision (currently most panels use the same `text-sm` for body and `text-2xl` for hero — a 6-step scale with consistent ratios would help), refined letter spacing on uppercase tracking labels, and considered use of accent color (currently navy is used for almost everything; a single warm accent for celebrations would create more visual interest).

### Intelligence — 86 (new dimension)
The app does a lot of smart things:
- **Anomaly detection** via 3σ-above-per-category-mean
- **Subscription detection** with type classification + price-change alerts
- **Categorization** via 200+ rule patterns + Ollama T3 fallback for uncategorized merchants
- **Fuzzy transaction dedup** at ingest time (just shipped)
- **Monte Carlo FIRE projection** seasoned with historical starting years
- **Net worth attribution** that splits debt paydown from real spending
- **Yield-arb suggestions** ranked by $-delta vs current setup
- **Card-rewards optimization** (best card for each merchant)
- **AI chat tool-use routing** — the LLM picks the right query function based on intent
- **Statement-close-day optimizer** for credit utilization timing
- **CFPB / state-AG redress matching** against transaction history
- **Stale-claim warnings** on Unclaimed (>30 days filed without payment)
- **Pattern detection** for recurring purchases across receipts

What's missing: no proactive notifications about *opportunities* (e.g., "you're about to hit Chase 5/24 — apply for Sapphire Preferred this week"), no AI-generated weekly summary in Notifications, no "you've been spending more on X lately" automatic insight surfacing.

### Completeness — 86 (strong)
Empty states everywhere, error states most places, deep-linkable via hash routing on every panel, skeletons on 16 panels and counting. The two remaining gaps:
- **Mobile parity** — only 6 of the 32 web panels have first-class mobile equivalents; the rest fall back to a desktop-shaped scaled view
- **Bulk operations** — most lifecycle states (activated/dismissed/snoozed/etc.) are per-row, no bulk select-all action

## Methodology caveats (same as last audit)

- **Code-based scoring is overconfident on Functionality** and underconfident on Delight/Beauty. A live-walk via Chrome MCP confirmed the visual lifts on the panels that were checked, but the chrome MCP timed out partway through this session (possibly the Inter font CDN handshake) so 8 panels weren't re-walked.
- The prior audit (#269) used live screenshots and is the better baseline for "how does this feel". The Δ column reflects my best estimate of how each score moved given the changes since.
- Live re-walk recommended before declaring 90 — many of these scores are within 5 of each other and code-only review can't disambiguate at that resolution.
- **Intelligence** is a new dimension — its 86 mean reflects what the app actually does, not what a user would notice in 5 minutes. Many of these features are passive (running in background) so they don't show up in a casual UX review.
