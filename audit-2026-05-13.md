## What's still imperfect

### 🟡 UX warts

1. **Gmail status aggregation is slow.** `/api/gmail/status` timed out at 12s during this audit on Chris's mailbox (~530 emails, 477 unmatched). The endpoint scans the full `email_messages` table for per-parser counts. **How to fix:** add a `(parser_name, parser_outcome)` composite index, or cache the rollup in a small status table.
2. **Cross-Store Deals scrapers** — Walmart ready, four others "needs auth" — same as the prior audit. Sprint 52's auth-expiry notification covers Albert; the deal scrapers don't yet emit notifications when their auth expires.
3. **Holdings preview is still the DEMO_DATA tile** — Plaid Investments product isn't granted. Sprint 39's stamp keeps it honest; the underlying upgrade path needs a separate effort.

### 🟢 Architectural

4. **No live screenshots in this audit pass** — Chrome MCP screenshots wedged on the slow `/api/gmail/status` calls. The text-and-axe verification path was sufficient, but a future audit should disable Gmail-status calls from page navigation (e.g. fetch on first Gmail-panel visit, not on every layout mount).
5. **`text-warn` darkened to `#8b5a00` for AA** — looks good on amber backgrounds and on white, but it shifts the brand-feel away from the orig Chase-yellow accent slightly. Worth confirming with Chris that the new tone reads OK before declaring done.
6. **Smoke-test inventory ships.** Sprint 52 added `smoke_balance_auth_notify.py`, Sprint 53 added `check_google_play_live.py`. Worth a one-time README sweep to register the new tests in `backend/README.md` smoke-test section.

1. **Gmail-status performance.** Add a `(parser_name, parser_outcome)` index on `email_messages`. One-line migration; the audit-probe path will go from 12s+ timeouts to milliseconds, and the Gmail panel + setup checklist will both feel snappier.

### Sprint-sized (1–3 hours)

3. **Cross-Store scraper auth-expiry notifications.** Reuse the Sprint 52 `notify.py` pattern for Target / Walmart / etc. Each gets registered under the same `scraper_auth_missing` notification kind; the existing `_KIND_META` mapping already routes them to Connections.
4. **Mobile vision-OCR button.** Port the Sprint 49 ReceiptsPanel button to ReceiptsScreen (React Native) and the corresponding mobile API helper.
5. **Shopping Patterns auto-feed from receipts.** When `ocr_vision_reparse` lands fresh line items, trigger the canonical-product matcher and update the Shopping Patterns rollup automatically. Closes the loop the audit identified for Intelligence.

### Bigger swing

6. **Source-citations in chat.** Chat answers cite the underlying transactions/subscriptions/receipts that backed the response. Touches `api/chat.py` + the Ask AI panel UI. Trust dimension's last remaining point.
7. **Keyboard + screen-reader walk-through.** Axe doesn't catch focus-order regressions or invisible-label issues. A manual pass (Tab through every panel, then VoiceOver/NVDA over every panel) is the only way to clear the last 0.5 point on Accessibility.
